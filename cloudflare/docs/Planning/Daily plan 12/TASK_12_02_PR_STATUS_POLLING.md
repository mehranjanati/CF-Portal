# TASK 12-02: PR Status Polling

## Objective
Implement a mechanism to track and display the status of GitHub Pull Requests and Actions.

## Requirements
- Poll the GitHub API for PR status and deployment status.
- Display status updates (e.g., "Pending", "Merging", "Success", "Failed") in the UI.
- Provide a link to the GitHub PR/Action for detailed view.

## Deliverables
- Polling logic in the frontend.
- UI component for displaying deployment status in `BuilderResultPanel`.
